## Get Started Fast

This guide will get you from zero to a working TokenStash in under 2 minutes.

<Steps>
  <Step title="Install the Plugin">
    Register the marketplace and install TokenStash:

    ```bash
    /plugin marketplace add abhinavdubeyad9/tokenstash
    /plugin install tokenstash@tokenstash
    ```

    <Info>
    You only need to do this once. The plugin persists across sessions.
    </Info>
  </Step>

  <Step title="Start a New Session">
    Start a new Claude Code session (or restart your current one).

    TokenStash automatically injects your stash at session start. If this is your first time, you'll see:

    ```
    TokenStash: No stash found. Run /stash-scan to index this project.
    ```

    <Note>
    The stash is loaded automatically via a `SessionStart` hook — you don't need to do anything.
    </Note>
  </Step>

  <Step title="Index Your Project (Optional)">
    If you want to populate the stash immediately with summaries of all files, run:

    ```bash
    /stash-scan
    ```

    This will:
    - Scan all files in your repository
    - Generate 3-5 line summaries for each file
    - Create an architecture overview
    - Save everything to `.tokenstash/`

    <Warning>
    Files over 500 lines are skipped during scanning. You can manually summarize them later if needed.
    </Warning>

    **Or skip this step:** The stash will build up automatically as you work on files.
  </Step>

  <Step title="Work on Your Project">
    Edit files normally. Every time you use the `Write` or `Edit` tool, TokenStash automatically:
    - Updates that file's summary in `.tokenstash/files/`
    - Keeps the stash in sync with your changes

    You don't need to do anything — it happens silently in the background.
  </Step>

  <Step title="Save Your Session Summary">
    At the end of a session (or whenever you make important decisions), run:

    ```bash
    /stash-save
    ```

    This saves:
    - Decisions made this session and why
    - Architecture changes (if any)
    - Summaries of any files you edited

    You'll see:

    ```
    TokenStash saved. Session summary written to .tokenstash/decisions.md.
    ```

    <Info>
    Your next session will start with all this context already loaded.
    </Info>
  </Step>
</Steps>

## What Happens Next?

The next time you start a Claude Code session:

1. TokenStash automatically loads your stash
2. Claude starts with full context of your project
3. No need to re-read files or re-explain architecture
4. You can immediately start coding

## Available Commands

Here's a quick reference of all TokenStash commands:

| Command | What it does |
|---------|-------------|
| `/stash-save` | Save session summary — decisions, architecture, file summaries |
| `/stash-scan` | Index the whole repo on demand |
| `/stash-scan --force` | Re-scan everything, overwrite existing summaries |
| `/stash-clear` | Wipe the project stash (asks for confirmation) |
