TokenStash organizes your project context into a simple directory structure. Here's what gets stashed and how it's formatted.

## Directory Layout

```
.tokenstash/
├── architecture.md    # High-level codebase overview
├── decisions.md       # Decision log (what and why)
├── rules.md           # Project-specific rules
└── files/             # Per-file summaries
    ├── src__index.ts.md
    ├── src__lib__utils.ts.md
    └── src__components__Button.tsx.md
```

<Info>
Path separators (`/`) are replaced with double underscores (`__`) in filenames.
</Info>

## Stash Locations

TokenStash supports two stash locations:

| Scope | Path | Priority | Use Case |
|-------|------|----------|----------|
| **Global** | `~/.claude/tokenstash/` | Base | Coding preferences, global rules |
| **Project** | `<repo-root>/.tokenstash/` | Override | Project-specific context |

### Priority Rules

- Project stash files **override** global files with the same name
- Global `preferences.md` and `rules.md` are loaded first
- Project `architecture.md`, `decisions.md`, and `rules.md` are loaded second
- File summaries are project-only (not global)

<Tabs>
  <Tab title="Global Stash">
    ```bash
    ~/.claude/tokenstash/
    ├── preferences.md   # Your coding style
    └── rules.md         # Universal rules (all projects)
    ```
    
    **Example `preferences.md`:**
    ```markdown
    - Prefer functional programming over classes
    - Use TypeScript strict mode
    - Write tests alongside features
    - Keep functions under 50 lines
    ```
  </Tab>
  
  <Tab title="Project Stash">
    ```bash
    my-app/.tokenstash/
    ├── architecture.md
    ├── decisions.md
    ├── rules.md
    └── files/
        ├── src__index.ts.md
        └── ...
    ```
    
    **Example `rules.md`:**
    ```markdown
    - All API routes must have rate limiting
    - Use Prisma for database access (no raw SQL)
    - Validate inputs with Zod schemas
    - Log errors to Sentry
    ```
  </Tab>
</Tabs>

## File Formats

### architecture.md

High-level codebase overview — structure, patterns, key components.

**Format:**

```markdown
## Architecture Overview
[2-4 sentences describing the overall structure]

## Key Components
- `path/to/module`: what it does
- `path/to/module`: what it does

## Patterns
- [notable patterns used]

## Tech Stack
- [main technologies]
```

**Real Example:**

```markdown
## Architecture Overview
Next.js 14 app using App Router, React Server Components, and TypeScript. 
Authentication via NextAuth.js with GitHub provider. Data stored in Postgres 
via Prisma ORM. Deployed on Vercel.

## Key Components
- `app/`: Next.js App Router pages and layouts
- `components/`: Reusable React components
- `lib/`: Database client, auth config, utilities
- `prisma/`: Database schema and migrations

## Patterns
- Server Components by default, Client Components for interactivity
- Server Actions for mutations
- Optimistic UI updates with useOptimistic

## Tech Stack
- Next.js 14, React 18, TypeScript
- Postgres, Prisma ORM
- NextAuth.js
- TailwindCSS
```

<Note>
Generated by `/stash-scan` and updated by `/stash-save` when structure changes.
</Note>

### decisions.md

Decision log — what was decided each session and why.

**Format:**

```markdown
## YYYY-MM-DD — <one-line session summary>

**Decisions:**
- [What was decided and why]
- [Another decision if applicable]

**Files changed:** file1.ts, file2.py
```

**Real Example:**

```markdown
## 2026-03-10 — Added real-time notifications

**Decisions:**
- Use Server-Sent Events instead of WebSockets (simpler, fewer resources)
- Poll database every 5s for new notifications (future: switch to Postgres LISTEN/NOTIFY)
- Cache read status in Redis (reduce DB load)

**Files changed:** app/api/notifications/route.ts, lib/redis.ts

## 2026-03-08 — Refactored authentication

**Decisions:**
- Migrate from JWT to session cookies (better security, easier rotation)
- Add CSRF protection with double-submit cookie pattern
- Implement refresh token rotation

**Files changed:** lib/auth.ts, middleware/verify.ts, app/api/auth/[...nextauth]/route.ts
```

<Tip>
Updated by `/stash-save` at the end of each session. Keep entries concise — focus on **why** over **what**.
</Tip>

### rules.md

Project-specific rules — CLAUDE.md on steroids.

**Format:**

Free-form markdown. Use bullet points for clarity.

**Real Example:**

```markdown
# Project Rules

## API Design
- All endpoints return `{ data, error }` format
- Use HTTP status codes: 200 (success), 400 (client error), 500 (server error)
- Validate inputs with Zod schemas
- Rate limit: 100 req/min per IP

## Database
- Use Prisma for all queries (no raw SQL)
- Enable soft deletes (deletedAt field)
- Index foreign keys
- Run migrations before deploy

## Testing
- Write unit tests for utilities
- Integration tests for API routes
- Use MSW for mocking HTTP requests
- Aim for 80% coverage

## Code Style
- Prefer named exports over default exports
- Use async/await over promises
- Keep components under 200 lines
- Extract magic numbers to constants
```

<Info>
Project `rules.md` overrides global `rules.md` if both exist.
</Info>

### files/*.md

Per-file summaries (3-5 lines each) — just enough context to decide if Claude needs to read the full file.

**Format:**

```markdown
**Purpose:** One sentence — what this file does.
**Key exports:** List the main functions/classes/constants exported.
**Patterns:** Any notable patterns (e.g. "uses factory pattern", "singleton").
**Last updated:** YYYY-MM-DD
```

**Real Examples:**

<Accordion title="src__lib__db.ts.md">
```markdown
**Purpose:** Prisma client singleton for database access.
**Key exports:** db (PrismaClient instance)
**Patterns:** Singleton pattern with global cache in dev mode.
**Last updated:** 2026-03-13
```
</Accordion>

<Accordion title="src__components__Button.tsx.md">
```markdown
**Purpose:** Reusable button component with variants and sizes.
**Key exports:** Button (React component)
**Patterns:** Uses class-variance-authority for style variants.
**Last updated:** 2026-03-12
```
</Accordion>

<Accordion title="src__middleware__auth.ts.md">
```markdown
**Purpose:** Authentication middleware for protected routes.
**Key exports:** requireAuth(), optionalAuth()
**Patterns:** Middleware factory pattern, returns Next.js middleware.
**Last updated:** 2026-03-11
```
</Accordion>

<Tip>
Keep summaries **under 5 lines**. Future Claude needs to know if they should read the file, not understand it fully.
</Tip>

## Path Flattening

File paths are flattened to create valid filenames:

| Original Path | Stash Filename |
|---------------|----------------|
| `src/index.ts` | `src__index.ts.md` |
| `src/lib/utils.ts` | `src__lib__utils.ts.md` |
| `app/api/users/route.ts` | `app__api__users__route.ts.md` |
| `components/ui/button.tsx` | `components__ui__button.tsx.md` |

**Reversal:**

From `hooks/session-start` (when loading summaries):

```bash
fname=$(basename "$f" .md | tr '__' '/')
```

<Info>
Double underscores (`__`) are converted back to slashes (`/`) when displaying filenames.
</Info>

## What Gets Excluded?

During `/stash-scan`, these are skipped:

<Accordion title="Directories">
- `.git/`
- `node_modules/`
- `.tokenstash/`
- `dist/`
- `build/`
- `.next/`
- `out/`
- `coverage/`
</Accordion>

<Accordion title="Files">
- Binary files (images, fonts, videos)
- Compiled artifacts (`.o`, `.pyc`, `.class`)
- Files over 500 lines (noted and skipped)
- Lock files (`package-lock.json`, `yarn.lock`)
</Accordion>

<Note>
You can manually add summaries for large files by creating `.tokenstash/files/<path>.md` yourself.
</Note>

## Size and Performance

Typical stash sizes:

| Project Size | File Count | Stash Size | Load Time |
|--------------|------------|------------|----------|
| Small (< 50 files) | 50 | ~10 KB | < 100ms |
| Medium (50-200 files) | 150 | ~30 KB | ~200ms |
| Large (200-1000 files) | 500 | ~100 KB | ~500ms |
| Very Large (1000+ files) | 1000+ | ~200 KB | ~1s |

<Tip>
200 KB of summaries is far cheaper than re-reading the entire codebase every session.
</Tip>

## Example: Full Stash

Here's what a complete stash looks like for a Next.js project:

```bash
my-app/.tokenstash/
├── architecture.md              # 15 lines
├── decisions.md                  # 50 lines (multiple sessions)
├── rules.md                      # 25 lines
└── files/                        # 147 files
    ├── app__layout.tsx.md
    ├── app__page.tsx.md
    ├── app__api__users__route.ts.md
    ├── components__Button.tsx.md
    ├── components__Header.tsx.md
    ├── lib__db.ts.md
    ├── lib__auth.ts.md
    ├── prisma__schema.prisma.md
    └── ...
```

**Total size:** ~95 KB (147 files × ~5 lines each + 90 lines of docs)

**Load time:** ~300ms at session start

**Value:** Claude knows the entire codebase structure without reading 147 files.
